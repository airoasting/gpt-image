# 채점 루브릭 (고정 — 채점자 공통)

사이트 갤러리의 각 프롬프트가 **실제 이미지를 얼마나 충실히 재현하는지**를 비전 판독으로 채점한다. 규칙을 취향이 아니라 실측으로 확정하기 위한 기준이다.

각 항목을 0~5로 채점해 `scores/gallery.scores.jsonl`에 1줄/1레코드로 기록:

```json
{"id":"ref-6","category":"웹툰","qa":{"goal_fit":0-5,"text_accuracy":0-5,"material_realism":0-5,"layout":0-5},"neg_rendered":bool,"notes":"<1문장>"}
```

- 합격선(스킬 게이트): qa 평균 ≥4 AND text_accuracy ≥4.
- 렌더 텍스트가 없는 컷은 `text_accuracy: null`로 두고 평균에서 제외.

## goal_fit — 프롬프트↔이미지 일치 (주지표)

5 = 프롬프트의 피사체·개수·구도·팔레트·무드가 이미지와 전부 일치. / 4 = 피사체·구도 일치, 무드/색 일부 이탈. / 3 = 피사체는 맞으나 구성 또는 개수 어긋남. / 1 = 다른 그림.

확인: 프롬프트가 요구한 요소가 **이미지에 전부 있는가**(없는 인물·사물 요구 여부), 이미지의 핵심 요소가 **프롬프트에 반영됐는가**.

## text_accuracy — 렌더 텍스트 정확도

5 = 모든 카피가 지시 문구 그대로, 각 1회, 지어낸 글자 0. / 4 = 전 글자 정확하나 스타일 흔들림. / 3 = 한 글자 오탈 또는 1회 중복. / 2 = 2+ 오탈 또는 유령 텍스트. / 1 = 절반 이상 훼손. / 0 = 식별 불가.

확인: 따옴표 카피의 받침·띄어쓰기, 중복 렌더, 프롬프트에 없는 텍스트 출현.

## layout — 영역·구성 정밀도

5 = 지시 영역 정확(타이틀 밴드가 상단, 번호 단계가 순서대로, 그리드 셀 수 정확). / 4 = 영역은 맞으나 마진/정렬 불균등. / 3 = 영역 경계 이탈. / 2 = 요소 위치 뒤바뀜·셀 수 오류. / 1 = 지시 대부분 무시.

## material_realism — 재질·질감

5 = 지시 질감(무광 세라믹·수채 번짐·벡터 평면 등) 사실적 + 인물은 natural skin texture. / 3 = 질감 평이·플라스틱. / 1 = 왁스 피부·AI 아티팩트.

## neg_rendered — 네거티브 렌더 여부 (정책 검증)

프롬프트의 배제 문구("~ 피합니다", "없이")가 이미지에 **글자나 요소로 렌더**됐는가. true면 "네거티브 대신 긍정형" 정책의 근거가 강해진다.

## 집계 (REPORT.md)

- 카테고리별·전체 qa 평균, 합격률(≥4).
- 텍스트 있는 컷의 text_accuracy 분포 → 캔버스 크기/따옴표 정책의 근거.
- neg_rendered 발생률 → 긍정형 정책 채택 근거.
- goal_fit이 낮은 컷 → 프롬프트-이미지 정합 재점검 대상.
